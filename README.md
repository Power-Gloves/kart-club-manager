# 超级马力卡丁车俱乐部 · 赛事计分系统 v2.0

## 完整赛制流程

```
① 车厂建车号池
② 车手报名（只填姓名）
③ 随机均分 A/B 两组 + 随机抽车
④ 交替进行（操作员可随时切换A/B页签）：
   A组排位 → A组预赛1 → B组排位 → B组预赛1 → A组预赛2 → B组预赛2
⑤ 裁判决定晋级人数（3~6人）→ 决赛
```

## 各阶段详解

### 排位赛（不计分，仅定发车顺序）
- 发车：随机顺序（车号已随机分配）
- 时长：8分钟自由跑
- 录入：每人最快圈时间（可选）或直接手动排序
- **结果用途**：决定预赛1的静态发车顺序

### 预赛1（计分）
- 发车：按排位赛结果静态起跑
- 圈数：10圈
- 录入：按实际完赛顺序点击车手
- 积分：N分给第1名，N-1给第2名…（N=组内人数）

### 预赛2（计分）
- 发车：**按预赛1成绩反转**（末位变头位）
- 换车：首尾对调车辆（消除车辆差异）
- 圈数：10圈
- 录入：按实际完赛顺序点击车手
- 积分规则同预赛1

### 总积分 = 预赛1 + 预赛2

### 决赛
- 晋级：裁判现场指定人数（看车辆状态，3~6人）
- 发车：按各组总积分降序排列
- 圈数：18圈，先冲线者获胜

## 换车/发车顺序逻辑（预赛2）

| 预赛1名次 | 预赛2发车位 | 预赛2使用的车 |
|----------|-----------|------------|
| 第1名     | 最后发车   | 原末位车手的车 |
| 第2名     | 倒数第2   | 原倒2车手的车  |
| 中间位次  | 中间       | 保持原车      |
| 倒数第2名 | 第2发车   | 原第2名的车   |
| 末位      | 第1发车   | 原第1名的车   |

## UI要求

- **手机端优先**：底部固定导航栏
- **可自由切换**：A组 / B组页签随时切换录入
- **数据持久化**：LocalStorage，刷新/切换页签不丢数据
- 每次操作立即保存

## 底部导航结构

```
[ 准备 ] [ A组 ] [ B组 ] [ 决赛 ]
```

### A组 / B组 内部子页签
```
[ 排位 ] [ 预赛1 ] [ 预赛2 ] [ 成绩 ]
```

## 技术方案
- 纯前端 HTML+CSS+JS，无服务器
- 双击 index.html 即可使用
- LocalStorage 持久化
